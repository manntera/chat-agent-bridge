# PoC（概念実証）計画：claude-discord-bridge

**プロジェクト名: `claude-discord-bridge`**

**本ドキュメントは PoC 範囲の計画を定義する。システム全体の設計は `01_SystemDesign.md` を参照。**

---

## 1. PoC の目的

本格実装に先立ち、システムの核心的な価値を最小構成で検証する。

> **PoC のゴール**
> スマートフォンの Discord から ClaudeCode に指示を送り、ClaudeCode が実際にコードを編集し、その結果が Discord に返ってくるという一連の体験が動作することを確認する。

---

## 2. PoC の範囲

複雑な機能をすべて削ぎ落とし、価値検証に必要な最小限の構成のみを実装する。

| PoC に含める | PoC では対応しない（本実装で対応） |
|-------------|----------------------------------|
| 通常のテキストチャンネルを使用 | フォーラムチャンネル + スレッド管理 |
| 単一セッション（1つの ClaudeCode プロセス） | 複数セッション / 並列実行 |
| テキスト送受信 + 2000文字分割 | フォルダ選択 UI（ドリルダウン） |
| 固定の作業ディレクトリ（`.env` で指定） | 会話フォーク（`!history` / `!fork`） |
| ユーザー ID 制限 + チャンネル ID 制限 | `!resume` / `!sessions` / `!status` |
| `!start` / `!stop` / `!interrupt` | Tailscale |
| 処理中の入力拒否 | |

---

## 3. システム構成

システム全体設計と同じ 3 層だが、各層を最小構成に絞る。

| 層 | PoC での構成 |
|----|-------------|
| **① スマートフォン** | Discord アプリ（そのまま利用） |
| **② Discord Bot** | Node.js + discord.js + node-pty。通常テキストチャンネルで動作 |
| **③ ClaudeCode** | `claude --dangerously-skip-permissions` を PTY 経由で起動 |

---

## 4. 環境変数

PoC では以下の環境変数のみを使用する。

| 環境変数 | 説明 |
|----------|------|
| `DISCORD_TOKEN` | Discord Bot の認証トークン |
| `CHANNEL_ID` | Bot が動作するテキストチャンネルの ID |
| `ALLOWED_USER_IDS` | 操作を許可する Discord ユーザー ID（カンマ区切り） |
| `WORK_DIR` | ClaudeCode の作業ディレクトリ（固定値） |

---

## 5. ドメインモデル

### 5.1 ドメインオブジェクト

#### Session（セッション）

システムの中核概念。ClaudeCode プロセスとの 1 対 1 の対話単位を表す。
PoC では同時に 1 つのセッションのみ存在できる。

**属性：**

| 属性 | 説明 |
|------|------|
| 状態（State） | セッションの現在の状態（下記の状態遷移を参照） |
| 作業ディレクトリ | ClaudeCode が操作するディレクトリ（`.env` の `WORK_DIR` で固定） |

**状態遷移：**

```
[セッション未存在]
    │
    │  !start
    ▼
  Idle（待機中）◄──────────┐
    │                      │
    │  ユーザー入力         │  応答完了
    ▼                      │
  Busy（処理中）───────────┘
    │
    │  !interrupt
    ▼
  Idle（待機中）
    │
    │  !stop
    ▼
[セッション未存在]
```

**状態ごとの振る舞い：**

| 状態 | ユーザー入力時の振る舞い | コマンドの受付 |
|------|--------------------------|----------------|
| セッション未存在 | `!start` 以外は「セッション未開始」と返す | `!start` のみ |
| Idle | 入力を ClaudeCode に転送し、Busy に遷移 | `!stop` |
| Busy | 「処理中です」と返し、入力を拒否 | `!interrupt`, `!stop` |

#### Command（コマンド）

Discord から届いたテキストを分類する概念。すべてのユーザー入力は、以下のいずれかに判定される。

| 種別 | トリガー | 説明 |
|------|----------|------|
| StartCommand | `!start` | セッションを開始する |
| StopCommand | `!stop` | セッションを終了する |
| InterruptCommand | `!interrupt` | 処理中のタスクを中断する（Ctrl+C 相当） |
| PromptInput | 上記以外のテキスト | ClaudeCode への入力として転送する |

**コマンドの有効性マトリクス：**

| コマンド | セッション未存在 | Idle | Busy |
|----------|------------------|------|------|
| `!start` | ✅ 実行 | ❌ 既に開始済み | ❌ 既に開始済み |
| `!stop` | ❌ 無効 | ✅ 実行 | ✅ 実行（中断して終了） |
| `!interrupt` | ❌ 無効 | ❌ 無効（処理なし） | ✅ 実行 |
| PromptInput | ❌ セッション未開始 | ✅ 転送 | ❌ 処理中 |

#### AccessControl（アクセス制御）

メッセージ処理の最初の関門。すべてのメッセージに対し、以下の条件を判定する。

| 条件 | 説明 |
|------|------|
| ユーザーID | `.env` の `ALLOWED_USER_IDS` に含まれるか |
| チャンネルID | `.env` の `CHANNEL_ID` と一致するか |

条件を満たさないメッセージは、一切の応答なく無視する（存在しないものとして扱う）。

#### BotResponse（Bot応答）

ClaudeCode の出力またはシステムメッセージを Discord に送信する責務を持つ。

**種別：**

| 種別 | 内容 |
|------|------|
| ClaudeCodeOutput | ClaudeCode からの応答テキスト |
| SystemMessage | セッション開始/終了/エラーなどのシステム通知 |
| RejectionMessage | 処理中の入力拒否、無効なコマンドなどの通知 |

**分割ルール：**

Discord のメッセージ上限（2000文字）を超える場合、複数メッセージに分割して送信する。

### 5.2 ドメインルール

**セッションの一意性** — PoC ではセッションは同時に 1 つのみ。`!start` でセッションが既に存在する場合は「既に開始済み」と返す。

**処理中の入力拒否** — セッションが Busy 状態のとき、PromptInput は拒否する。ただし `!interrupt` と `!stop` は受け付ける。

**アクセス制御の優先** — アクセス制御の判定は、コマンド解析やセッション状態の確認よりも先に行う。許可されないユーザーからのメッセージは、コマンドの種別に関わらず無視する。

### 5.3 メッセージ処理フロー

ユーザーからのメッセージが処理される全体フローを以下に示す。

```
Discord メッセージ受信
    │
    ▼
[AccessControl] ── 拒否 → 無視（応答なし）
    │ 許可
    ▼
[Command判定] ── コマンド種別を特定
    │
    ├─ StartCommand ──→ セッション未存在なら開始 / 存在するならエラー応答
    ├─ StopCommand ───→ セッション存在なら終了 / 未存在ならエラー応答
    ├─ InterruptCommand → Busy なら中断 / それ以外はエラー応答
    └─ PromptInput ───→ [Session状態チェック]
                            ├─ 未存在 → 「セッション未開始」応答
                            ├─ Idle   → ClaudeCodeに転送、Busyに遷移
                            └─ Busy   → 「処理中」応答（入力拒否）
```

## 6. 完了条件

以下のシナリオがスマートフォンから動作することをもって PoC 完了とする。

1. `!start` でセッションが開始される
2. Discord でテキストを送信すると、ClaudeCode が応答を返す
3. 「このファイルにテスト追加して」のような指示で、実際にコードが編集される
4. 長い応答が 2000 文字ごとに分割されて送信される
5. 処理中に入力すると「処理中です」と返される
6. `!interrupt` で処理中のタスクが中断される
7. `!stop` でセッションが終了される
