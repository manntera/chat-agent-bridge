# 設計レポート：スマートフォンから ClaudeCode を遠隔操作するシステム

**プロジェクト名: `claude-discord-bridge`**

**Discord Forum × ClaudeCode Bridge**

最終更新: 2025年

---

## 1. 背景と課題

### 1.1 現在の開発環境

開発者は自宅の Ubuntu マシンで ClaudeCode（AI アシスタント付きコーディングツール）を使って開発作業を行っている。ClaudeCode は AI が自律的にコードを読み書き・実行できるツールであり、複雑なプログラミングタスクを対話形式でこなすことができる。

### 1.2 課題

> **問題**
> ClaudeCode は Ubuntu マシン上でしか操作できないため、外出中や移動中にスマートフォンからアクセスする手段がなく、開発作業を継続できない。

**具体的な不便：**

- 外出先では開発作業が完全に止まってしまう
- 思いついたアイデアをすぐに試せない
- 緊急のバグ修正などに対応できない

---

## 2. 目標

### 2.1 主目標

> **ゴール**
> スマートフォンから Discord のチャット画面を操作するだけで、自宅の Ubuntu マシンで動く ClaudeCode を遠隔操作し、外出先でも開発作業を継続できるようにする。

### 2.2 追加要件

単純な遠隔操作にとどまらず、以下の高度な操作も実現する。

| 機能 | 説明 |
|------|------|
| **コンテキスト管理** | 複数の作業（機能開発・バグ修正など）をそれぞれ独立した「会話」として管理する |
| **過去の呼び出し** | 過去に行った作業の会話を中断した状態から再開できる |
| **会話のフォーク** | 過去の任意の時点に遡り、そこから別のアプローチを試す分岐ができる（Git のブランチのような概念） |
| **並列作業** | 複数の作業を同時に進め、それぞれ独立して操作できる |

## 3. 技術的な決定事項

複数の候補を比較検討した結果、以下の通り決定した。

| 項目 | 決定内容 |
|------|----------|
| **アプローチ** | Discord Bot ブリッジ（Discord フォーラム × Bot） |
| **ClaudeCode との通信** | サブプロセスモード（`claude -p`）で claude CLI をメッセージ単位で実行 |
| **権限確認** | `--dangerously-skip-permissions` で自動承認 |
| **実装言語** | TypeScript / Node.js（discord.js + child_process） |
| **長い出力の処理** | 2000文字で分割して複数 Discord メッセージとして送信 |
| **同時セッション数** | 上限なし（マシンリソース次第） |
| **作業ディレクトリ** | スレッド作成時に番号入力式のフォルダ選択UIで指定（`.env` の `PROJECTS_ROOT` が起点） |
| **会話履歴の保存** | ClaudeCode の JSONL ファイルを直接操作（`~/.claude/projects/` 配下） |

### 3.1 アプローチの採用理由

複数の候補（SSH + tmux、Tailscale + VSCode Remote など）を比較検討した結果、「Discord Bot ブリッジ」を採用する。

- スマートフォンから普通のチャット感覚で操作できる
- Discord のフォーラム機能がセッション管理と自然にマッピングできる
- セッションのフォーク（分岐）や並列実行を直感的な操作で実現できる

### 3.2 サブプロセス通信方式の詳細

Bot は Node.js の **child_process** モジュールを使い、ユーザーのメッセージごとに **claude** CLI を非対話モード（`-p`）で起動する。`--session-id` オプションにより会話の継続性が保たれるため、プロセスを常駐させる必要がない。

**起動コマンドの例：**

```bash
claude -p "このファイルにテスト追加して" \
  --session-id <セッションID> \
  --output-format stream-json \
  --dangerously-skip-permissions
```

**サブプロセス方式の通信フロー：**

1. Discord でユーザーがメッセージを送信
2. Bot がメッセージを受信し、`claude -p` をサブプロセスとして起動（`--session-id` で会話を継続）
3. ClaudeCode が処理を実行し、構造化 JSON を stdout にストリーミング出力
4. Bot が途中経過（ツール使用・拡張思考）を検知するたびに Discord にリアルタイム送信
5. プロセスが終了すると応答完了（exit code で成功/失敗を判定）
6. Bot が最終結果を Discord スレッドに返信（必要に応じ2000文字で分割）

**サブプロセス方式の利点：**

| 観点 | 説明 |
|------|------|
| 応答完了の検知 | プロセス終了 = 応答完了。ヒューリスティクスな判定が不要 |
| 出力形式 | `--output-format stream-json` により構造化 JSON で取得。ANSI エスケープシーケンスの除去が不要 |
| セッション継続 | `--session-id` で会話履歴が自動復元される。プロセスを常駐させる必要がない |
| 中断操作 | `SIGINT` シグナル送信で中断。PTY へのバイト送信のような低レベル操作が不要 |
| エラー検知 | exit code + stderr で確実に判定 |
| 並列実行 | 独立したプロセスとして自然に並列化。複数セッションの管理が容易 |

### 3.3 環境変数による設定（.env）

Bot の起動時に `.env` ファイルから各種パスや設定値を読み込む。以下が主な環境変数である。

| 環境変数 | 説明 |
|----------|------|
| `DISCORD_TOKEN` | Discord Bot の認証トークン |
| `FORUM_CHANNEL_ID` | Bot が動作するフォーラムチャンネルの ID |
| `ALLOWED_USER_IDS` | 操作を許可する Discord ユーザー ID（カンマ区切り） |
| `PROJECTS_ROOT` | フォルダ選択UIの起点となるパス（例: `/home/user/projects`） |
| `CLAUDE_PATH` | claude CLI のパス（デフォルト: `claude`） |

## 4. システム構成

### 4.1 3層アーキテクチャ

システムは以下の 3 層で構成される。

| 層 | 役割 | 技術要素 |
|----|------|----------|
| **①** | スマートフォン — Discord アプリからメッセージを送受信するだけ。特別なアプリ不要 | Discord モバイルアプリ |
| **②** | Discord Bot（Ubuntu 上で常時起動）— メッセージを受け取り、ClaudeCode へ転送。結果を返信する仲介役 | Node.js + discord.js + child_process |
| **③** | ClaudeCode（Ubuntu 上で動作）— コード生成・編集・実行を行う AI ツール | claude CLI (`-p` サブプロセスモード) |

### 4.2 Discord フォーラムとセッションの対応関係

> **セッション（Session）の定義**
> ClaudeCode との一連の会話を指す。`--session-id` によって一意に識別される。1 つのセッションは 1 つの Discord フォーラムスレッドと対応する。

Discord の「フォーラムチャンネル」の機能を活用し、作業ごとにセッションを分離・管理する。

| Discord の概念 | ClaudeCode での意味 |
|----------------|---------------------|
| フォーラムの投稿（スレッド） | 1 つのセッション（例：「機能 A 開発」） |
| スレッド内の返信 | そのセッションへの入力・出力 |
| 新しいスレッドを作成 | 新しいセッションを開始 |
| スレッドのフォーク | 会話履歴の任意の時点から分岐 |
| 複数スレッドを同時に操作 | 複数のセッションを並列実行 |

### 4.3 スレッド作成時の作業ディレクトリ選択

新しいスレッドを作成すると、まず作業ディレクトリを選択するインタラクティブなフローが開始される。フォルダ一覧が番号付きで表示され、番号を入力して階層を深くしていく。目的のフォルダに達したら「決定」でセッションが開始される。

**環境変数による設定：**

`.env` ファイルに `PROJECTS_ROOT` を設定し、フォルダ選択の起点とする。Bot 起動時にこの値が読み込まれる。

```
PROJECTS_ROOT=/home/user/projects
```

**フォルダ選択の操作フロー（例）：**

> ```
> Bot: 📁 作業ディレクトリを選択してください
> /home/user/projects/
>  1. webapp
>  2. api-server
>  3. mobile-app
>  0: ✅ ここに決定 | ..: ⬆️ 戻る
>
> User: 1
>
> Bot: 📁 /home/user/projects/webapp/
>  1. frontend
>  2. backend
>  3. shared
>  0: ✅ ここに決定 | ..: ⬆️ 戻る
>
> User: 0
>
> Bot: ✅ 作業ディレクトリ: /home/user/projects/webapp/
> セッションを開始します...
> ```

**フォルダ表示のフィルタ：**

- 隠しフォルダ（`.` で始まるもの）は非表示
- `node_modules`、`.git`、`__pycache__` などの開発生成物は非表示
- ディレクトリのみ表示（ファイルは非表示）

## 5. 会話フォーク機能の詳細

### 5.1 フォークとは

「フォーク」とは、進行中の会話を過去の任意の時点まで巻き戻し、そこから新しい別の方向に進める機能である。ソフトウェア開発における Git のブランチ機能に相当する。

> **具体的なユースケース**
> 「JWT で認証を実装して」と指示した後、実装が完成したが、やはり「OAuth でやり直したい」となった場合、JWT の指示を出す前の時点に戻り、そこから「OAuth で実装して」と新たな分岐を作れる。元の JWT 実装の会話はそのまま残る。

### 5.2 仕組み

ClaudeCode は会話の全履歴を JSONL 形式（1 行 1 メッセージのテキストファイル）で保存している。この特性を利用し、以下の手順でフォークを実現する。

1. `!history` コマンドで現在の会話履歴を番号付きで表示
2. 戻りたいメッセージの番号を確認
3. `!fork 2 新タイトル` と入力（例：2番以降を切り捨て）
4. 指定した時点までの履歴のみをコピーした新セッションが自動作成
5. 新しい Discord スレッドが自動生成され、そこから別の指示を送れる

## 6. 操作コマンド一覧

Discord のスレッド内で以下のコマンドを入力することで、各操作を行える。

| コマンド | 操作内容 |
|----------|----------|
| （新しいスレッドを作成） | 新しいセッションを開始。まずフォルダ選択UIで作業ディレクトリを指定 |
| `!resume` | 停止中のセッションを再開 |
| `!stop` | セッションを停止（履歴は保持） |
| `!history` | 会話履歴を番号付きで表示 |
| `!fork <番号> [タイトル]` | 指定番号の時点から会話を分岐 |
| `!sessions` | 全セッションの一覧と起動状態を表示 |
| `!status` | 現在のセッションの状態を確認 |
| `!interrupt` | 実行中の処理を中断（SIGINT 送信） |
| （上記以外のテキスト） | そのまま ClaudeCode への入力として転送 |

## 7. セキュリティ

自宅マシンを外部から操作するにあたり、以下のセキュリティ対策を実施する。

| 対策 | 内容 |
|------|------|
| **Bot 自身の無視** | Bot 自身が送信したメッセージを無視する（無限ループ防止） |
| **ユーザー制限** | 許可した Discord ユーザー ID のみが操作可能。他のユーザーのメッセージは完全に無視 |
| **チャンネル制限** | 指定したフォーラムチャンネル内でのみ Bot が動作 |
| **ネットワーク** | Tailscale（VPN）を利用してポート開放なしで安全に接続することを推奨 |
