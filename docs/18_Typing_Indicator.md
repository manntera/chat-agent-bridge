# 18. Typing Indicator（入力中インジケーター）

## 背景

Discord 側からAIが「考えている最中」なのか「落ちた状態」なのか区別できない。
ユーザーが処理状況を把握できる仕組みが必要。

## 検討した案

### 案1: Typing Indicator（タイピング表示）
- `channel.sendTyping()` で「Botが入力中...」を表示
- 10秒で自動消滅するため、定期的に再送が必要
- プロセスが落ちた場合はタイピング表示が自然に消えるため、死活監視として機能する

### 案2: リアクションベース
- 処理開始時にユーザーのメッセージに ⏳ リアクションを付与
- 処理完了時に除去（or ✅ に変更）

### 案3: 案1+案2の組み合わせ

## 結論

**案1（Typing Indicator）を採用する。**

理由:
- 10秒で消える性質がヘルスチェックとして機能する
- プロセスが生きている間は表示が継続し、落ちたら自然に消えるため「落ちたのか、長時間思考中なのか」を区別できる
- リアクションベースでは落ちた場合と思考中の場合の区別がつかない

## 現状の問題

以前に `sendTyping()` の実装が存在したが、ユーザーが削除した。
これは Discord API の仕様上、**メッセージ送信時に typing インジケーターが自動クリアされる**ため、正しく機能していなかったことが原因。

具体的な問題:
1. `startTyping()` で typing 表示開始
2. 直後に `sendText('📨 受信しました...')` を送信 → **typing クリア**
3. 8秒後にインターバルで `sendTyping()` → typing 表示
4. progress メッセージ（`🔧 Edit: ...` 等）送信 → **typing クリア**

メッセージ送信のたびにタイピング表示が消え、次の8秒インターバルまで表示されない。

## 実装方針

`sendText()` / `sendEmbed()` でメッセージ送信後、typing 状態であれば `sendTyping()` を再送する。

### 変更対象
- `ThreadSender` インターフェースに `sendTyping()` を再追加
- `createNotifier` 内で typing のインターバル管理を再実装
- メッセージ送信後の `sendTyping()` 再送ロジックを追加

### タイミング
- **開始**: `progress` の `started` イベント受信時
- **継続**: 8秒間隔 + メッセージ送信後
- **停止**: `usage` 通知受信時（処理完了）
