# ドメインモデル：claude-discord-bridge PoC

**スコープ: PoC（概念実証）範囲のみ**

---

## 1. ドメインモデル概要

本ドキュメントは、PoCにおけるシステムの「何をやるか」をドメインモデルとして定義する。
技術的な実装手段（PTYの扱い、ANSIエスケープ処理など）は本ドキュメントの対象外とし、別途システム設計で定める。

### 1.1 PoCで実現すること

- Discordのテキストチャンネルを通じて、ClaudeCodeと対話する
- セッションの開始・終了・中断を明示的なコマンドで制御する
- 許可されたユーザーのみが操作できる
- ClaudeCodeの出力をDiscordの制約に合わせて分割・送信する

### 1.2 PoCで実現しないこと

- フォーラムチャンネル / スレッド管理
- 複数セッション / 並列実行
- 会話フォーク（`!history` / `!fork`）
- フォルダ選択UI（ドリルダウン）
- `!resume` / `!sessions` / `!status`

## 2. ドメインオブジェクト

### 2.1 Session（セッション）

システムの中核概念。ClaudeCodeプロセスとの1対1の対話単位を表す。
PoCでは同時に1つのセッションのみ存在できる。

**属性：**

| 属性 | 説明 |
|------|------|
| 状態（State） | セッションの現在の状態（後述の状態遷移を参照） |
| 作業ディレクトリ | ClaudeCodeが操作するディレクトリ（`.env` の `WORK_DIR` で固定） |

**状態遷移：**

```
[セッション未存在]
    │
    │  !start
    ▼
  Idle（待機中）◄──────────┐
    │                      │
    │  ユーザー入力         │  応答完了
    ▼                      │
  Busy（処理中）───────────┘
    │
    │  !interrupt
    ▼
  Idle（待機中）
    │
    │  !stop
    ▼
[セッション未存在]
```

**状態ごとの振る舞い：**

| 状態 | ユーザー入力時の振る舞い | コマンドの受付 |
|------|--------------------------|----------------|
| セッション未存在 | `!start` 以外は「セッション未開始」と返す | `!start` のみ |
| Idle | 入力をClaudeCodeに転送し、Busyに遷移 | `!stop` |
| Busy | 「処理中です」と返し、入力を拒否 | `!interrupt` |

### 2.2 Command（コマンド）

Discordから届いたテキストを分類する概念。すべてのユーザー入力は、以下のいずれかに判定される。

| 種別 | トリガー | 説明 |
|------|----------|------|
| StartCommand | `!start` | セッションを開始する |
| StopCommand | `!stop` | セッションを終了する |
| InterruptCommand | `!interrupt` | 処理中のタスクを中断する（Ctrl+C相当） |
| PromptInput | 上記以外のテキスト | ClaudeCodeへの入力として転送する |

**コマンドの有効性ルール：**

| コマンド | セッション未存在 | Idle | Busy |
|----------|------------------|------|------|
| `!start` | ✅ 実行 | ❌ 既に開始済み | ❌ 既に開始済み |
| `!stop` | ❌ 無効 | ✅ 実行 | ✅ 実行（中断して終了） |
| `!interrupt` | ❌ 無効 | ❌ 無効（処理なし） | ✅ 実行 |
| PromptInput | ❌ セッション未開始 | ✅ 転送 | ❌ 処理中 |

### 2.3 AccessControl（アクセス制御）

メッセージ処理の最初の関門。すべてのメッセージに対し、以下の条件を満たすかを判定する。

| 条件 | 説明 |
|------|------|
| ユーザーID | `.env` の `ALLOWED_USER_IDS` に含まれるか |
| チャンネルID | `.env` の `CHANNEL_ID` と一致するか |

条件を満たさないメッセージは、一切の応答なく無視する（存在しないものとして扱う）。

### 2.4 BotResponse（Bot応答）

ClaudeCodeの出力またはシステムメッセージをDiscordに送信する責務を持つ。

**種別：**

| 種別 | 内容 |
|------|------|
| ClaudeCodeOutput | ClaudeCodeからの応答テキスト |
| SystemMessage | セッション開始/終了/エラーなどのシステム通知 |
| RejectionMessage | 処理中の入力拒否、無効なコマンドなどの通知 |

**分割ルール：**

Discordのメッセージ上限（2000文字）を超える場合、複数メッセージに分割して送信する。

## 3. ドメインルール

### 3.1 セッションの一意性

PoCではセッションは同時に1つのみ。`!start` でセッションが存在する場合は「既に開始済み」と返す。

### 3.2 処理中の入力拒否

セッションがBusy状態のとき、PromptInputは拒否する。ただし `!interrupt` と `!stop` は受け付ける。

### 3.3 アクセス制御の優先

アクセス制御の判定は、コマンド解析やセッション状態の確認よりも先に行う。許可されないユーザーからのメッセージは、コマンドの種別に関わらず無視する。

---

## 4. メッセージ処理フロー

ユーザーからのメッセージが処理される流れを以下に示す。

```
Discord メッセージ受信
    │
    ▼
[AccessControl] ── 拒否 → 無視（応答なし）
    │ 許可
    ▼
[Command判定] ── コマンド種別を特定
    │
    ├─ StartCommand ──→ セッション未存在なら開始 / 存在するならエラー応答
    ├─ StopCommand ───→ セッション存在なら終了 / 未存在ならエラー応答
    ├─ InterruptCommand → Busy なら中断 / それ以外はエラー応答
    └─ PromptInput ───→ [Session状態チェック]
                            ├─ 未存在 → 「セッション未開始」応答
                            ├─ Idle   → ClaudeCodeに転送、Busyに遷移
                            └─ Busy   → 「処理中」応答（入力拒否）
```

## 5. 環境変数（PoC）

| 環境変数 | 説明 |
|----------|------|
| `DISCORD_TOKEN` | Discord Bot の認証トークン |
| `CHANNEL_ID` | Bot が動作するテキストチャンネルの ID |
| `ALLOWED_USER_IDS` | 操作を許可する Discord ユーザー ID（カンマ区切り） |
| `WORK_DIR` | ClaudeCode の作業ディレクトリ（固定値） |
